Fullstack-приложение для взаимодействия с пользователями по профессиям, личным качествам и через систему комментариев.

Приложение построено на MERN-стеке и демонстрирует современный подход к разработке клиент-серверных приложений с использованием React, TypeScript, RTK Query, MongoDB, Express и Docker.

## О проекте

Пользователи могут регистрироваться, просматривать профили, оставлять комментарии, редактировать собственные данные, а также использовать поиск, фильтрацию и сортировку.

Приложение реализовано в качестве Single Page Application (SPA) с REST API и удаленной базой данных MongoDB.

## Основные возможности

### Аутентификация и авторизация

-   регистрация пользователей;
-   авторизация пользователей;
-   JWT-аутентификация;
-   access token и refresh token;
-   безопасное хранение паролей с использованием bcrypt;
-   защищенные маршруты.

### Работа с пользователями

-   просмотр таблицы пользователей;
-   просмотр профиля пользователя;
-   редактирование собственного профиля;
-   автоматическая генерация аватаров через DiceBear;
-   выбор профессии;
-   управление личными качествами.

### Поиск и навигация

-   поиск пользователей;
-   фильтрация пользователей;
-   сортировка пользователей;
-   пагинация таблицы пользователей.

### Комментарии

-   просмотр комментариев;
-   создание комментариев;
-   удаление комментариев.

### Интерфейс

-   светлая тема;
-   темная тема;
-   переключение темы оформления;
-   современный UI на базе Ant Design.

### Автоматическая инициализация данных

При первом успешном подключении к базе данных приложение автоматически создает:
-   список профессий;
-   список качеств.

## Управление состоянием

Для управления состоянием используется комбинированный подход.

### Redux Toolkit

Используется для:
-   хранения состояния авторизации;
-   управления статусом аутентификации;
-   хранения служебных данных пользователя.

### RTK Query

Используется для:
-   выполнения HTTP-запросов;
-   кэширования данных;
-   синхронизации серверного состояния;
-   автоматического обновления данных.

| Маршрут | Описание |
|----------|----------|
| `/` | Главная страница с таблицей пользователей |
| `/users/:userId` | Просмотр профиля пользователя |

## Запуск проекта локально

### Требования

Перед запуском убедитесь, что установлены:
-   Node.js;
-   npm.

### Клонирование репозитория

Через HTTPS:

```bash
git clone https://github.com/develengie/meet-fs-app.git
cd meet-fs-app
```

Через SSH:

```bash
git clone git@github.com:develengie/meet-fs-app.git
cd meet-fs-app
```

### Установка зависимостей Backend-а

```bash
cd backend
npm install
```

### Установка зависимостей Frontend-а

```bash
cd ../frontend
npm install
```

### Запуск Backend-а

```bash
cd backend
npm run serve
```

Backend будет доступен по адресу:

```text
http://localhost:8080
```

### Запуск Frontend-а

```bash
cd frontend
npm run dev
```

Frontend будет доступен по адресу:

```text
http://localhost:5173
```

## Запуск проекта через Docker

Docker-образ выполняет production-сборку frontend-приложения, устанавливает зависимости backend-а и запускает Express-сервер, который обслуживает как REST API, так и клиентское приложение.

### Сборка образа

```bash
docker build -t meet-fs-app .
```

### Запуск контейнера

```bash
docker run -d -p 8080:8080 --name meet-fs-app --rm meet-fs-app
```

После запуска приложение будет доступно по адресу:

```text
http://localhost:8080
```

## Production

В production-режиме:

1.  React-приложение собирается в директорию `frontend/dist`.
2.  Express раздает статические файлы из `frontend/dist`.
3.  Все API-запросы обрабатываются через `/api`.
4.  Для поддержки клиентской маршрутизации возвращается `index.html`.
